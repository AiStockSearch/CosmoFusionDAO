# Переводы CosmoFusion DAO

Этот каталог содержит объединенные JSON файлы со всеми переводами сайта CosmoFusion DAO.

## Файлы переводов

- `translations-en.json` - Английские переводы
- `translations-ru.json` - Русские переводы

## Структура файлов

Каждый файл содержит следующие разделы:

### Основные разделы
- `hero` - Главная секция с заголовком, описанием и призывом к действию
- `problem` - Секция проблем и вызовов
- `solution` - Секция решений и подходов
- `reflection` - Секция размышлений и инсайтов
- `governance` - Секция управления и принятия решений
- `evolution` - Секция эволюции и развития
- `gettingStarted` - Секция "С чего начать"
- `motionPhrase` - Фраза движения
- `footer` - Подвал сайта
- `journalEntries` - Записи в журнале
- `seo` - SEO метаданные для каждой секции

### Структура каждой секции

```json
{
  "title": "Заголовок секции",
  "subtitle": "Подзаголовок (если есть)",
  "description": "Описание секции",
  "intro": "Вводный текст",
  "problems": [
    {
      "title": "Название проблемы",
      "description": "Описание проблемы"
    }
  ],
  "quote": "Цитата или ключевая фраза"
}
```

## Использование

### Импорт в React компонентах

```typescript
import enTranslations from './translations-en.json';
import ruTranslations from './translations-ru.json';

// Получение перевода по языку и секции
const getTranslation = (language: 'en' | 'ru', section: string, key: string) => {
  const translations = language === 'en' ? enTranslations : ruTranslations;
  return translations[section]?.[key] || '';
};

// Пример использования
const heroTitle = getTranslation('en', 'hero', 'title');
```

### Обновление переводов

1. Откройте соответствующий JSON файл
2. Найдите нужную секцию и ключ
3. Измените значение
4. Сохраните файл

### Добавление новых переводов

1. Добавьте новый ключ в обе языковые версии
2. Убедитесь, что структура одинакова в обоих файлах
3. Проверьте корректность JSON синтаксиса

### Валидация

Перед коммитом убедитесь, что:
- JSON файлы корректно форматированы
- Все ключи присутствуют в обеих языковых версиях
- Нет синтаксических ошибок

## Преимущества объединенных файлов

1. **Централизованное управление** - все переводы в одном месте
2. **Легкое редактирование** - простое изменение текстов
3. **Структурированность** - четкая организация по секциям
4. **Масштабируемость** - легко добавлять новые языки
5. **Версионность** - отслеживание изменений в Git

## Миграция с существующих файлов

Старые файлы переводов (`heroPageEn.tsx`, `footerRu.tsx` и т.д.) можно постепенно заменить на использование этих JSON файлов. Это упростит управление переводами и сделает процесс редактирования более удобным.
